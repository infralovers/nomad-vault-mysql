Protect your data!


The Transit secrets engine in HashiCorp Vault handles cryptographic functions on data in transit. Vault does not store the data sent to the secrets engine.




The primary use case for Transit is to encrypt data from applications while still storing that encrypted data in some primary data store.

Think of a scenario where an organisation must cryptographically protect personally identifiable information (PII) while preserving data format and length.

In this demonstration, transit protects sensitive customer fields before they are stored in MySQL.
